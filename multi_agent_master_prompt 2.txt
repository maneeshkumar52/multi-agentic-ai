PROJECT OVERVIEW:
Build a complete ENTERPRISE-GRADE multi-agent pipeline with Streamlit frontend, web 
search, compliance validation, document formatting (Markdown/PDF/Word), and email 
delivery. Uses Azure OpenAI for AI summaries and SMTP for email delivery.

MULTI-AGENT ARCHITECTURE (5 AGENTS):
1. WebSearchAgent - DuckDuckGo scraping with Azure OpenAI summaries
2. ComplianceGuardAgent - Citation validation, hallucination control, quality checks
3. DocumentFormatterAgent - Generate MD/PDF/DOCX with validated content
4. EmailerAgent - Send professional emails with attachments
5. PipelineOrchestrator - Coordinate all agents sequentially

PIPELINE FLOW:
Streamlit UI → Orchestrator → WebSearch → ComplianceGuard → DocumentFormatter → EmailerAgent

TECH STACK: 
Python 3.8+, Streamlit, Azure OpenAI, SMTP, PyMuPDF (for PDF), python-docx, 
markdown2, beautifulsoup4, requests

CRITICAL IMPLEMENTATION RULES (APPLY TO ALL PHASES):
✓ Use PyMuPDF (fitz) for PDF generation - works on Windows without GTK dependencies
✓ Azure OpenAI: Initialize WITHOUT 'proxies' parameter to avoid errors
✓ Always check None/null before Path() or file operations
✓ Filter None values from attachment lists before processing
✓ Use modern EmailMessage API (NOT MIMEMultipart) per Python 3.12+ standards
✓ Singleton pattern for loggers to avoid duplicate handlers
✓ Graceful error handling with try/except and informative logging
✓ Validate configuration before executing pipeline
✓ Install ALL dependencies at once in Phase 1 (frontend + backend)
✓ Run frontend and backend in SEPARATE terminal windows to avoid conflicts
✓ CRITICAL: ComplianceGuardAgent MUST validate all content before document generation
✓ Pipeline MUST stop if ComplianceGuard rejects content (approved=False)

AGENT EXECUTION ORDER (STRICT SEQUENCE):
1. WebSearchAgent → Scrape DuckDuckGo
2. ComplianceGuardAgent → Validate citations, remove duplicates, check quality
